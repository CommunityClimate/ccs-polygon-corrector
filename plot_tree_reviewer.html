📖 Plot Tree Reviewer — User Handbook
1 — Getting Started
Connecting to Fieldforce
Click ☁ Connect to Fieldforce in the left sidebar. You will be prompted to sign in with your Microsoft account. Once authenticated, the system loads all plots from Dataverse automatically — no file upload needed.
After connecting, a dropdown appears listing all available plots. Use the search box above the dropdown to type part of a plot name and filter the list instantly.
What loads automatically
When you select a plot, the system fetches its polygon boundary, RefCenter, and MeasuredCenter coordinates directly from the plot record in Fieldforce. No copy-paste required.
💡 If the connection times out, click the ↺ Refresh button next to the Connect button to reload the current plot without re-authenticating.
2 — Selecting a Plot
Using the dropdown
After connecting, type in the search box to filter by plot name, then click a plot in the dropdown. The map, polygon, and all coordinates populate instantly.
Uploading trees manually
If you have a tree export as an .xlsx or .csv file, drag it into the upload zone below the OR UPLOAD divider. Column names (lat, lng, tree UID) are detected automatically. If columns cannot be matched, a manual picker appears.
Pasting coordinates
You can also paste tree coordinates directly into the Tree Coordinates text box — one coordinate pair per line in lat,lng format. Click ▶ Run Analysis to process.
💡 Use the ⊞ Plot Catalog button in the header to browse all plots visually — then click any card to jump straight to that plot.
3 — Reference & Measured Center
RefCenter (GPS Survey)
The RefCenter is the GPS reference point recorded during the formal survey of the plot. It is plotted as a blue dot on the map. The system measures how far each tree is from this point.
MeasuredCenter (Field Officer)
The MeasuredCenter is the GPS point recorded by the field officer at the centre of the plot at planting time. It is plotted as a purple dot. This often differs slightly from the RefCenter — the offset between the two is shown in the measurements panel.
Switching the reference source
Use the radio buttons at the top of the Reference Point section to choose which point is used as the reference when counting inside/outside trees: RefCenter, MeasuredCenter, or None. This does not affect the polygon boundary — only which centre point is displayed and used for distance calculations.
⚠ If RefCenter and MeasuredCenter are far apart, this usually indicates a data entry error or the polygon was drawn at the wrong location. Check both points on the map before correcting.
4 — Map & Trees
Reading the map
The map shows a satellite view of the plot area. The green dashed polygon is the registered plot boundary. Trees are shown as numbered circles — green if inside the polygon, red if outside.
Tree list panel
The panel on the right shows all trees with their status, coordinates, and UID. Click any tree in the list to highlight it on the map and jump to its location.
Side lengths
Click 📐 Side Lengths to label every polygon edge with its length in metres. Green = within target range, red = significantly off. A summary shows average side length, total perimeter, and estimated area.
A→B Ruler
Click 📏 A→B Ruler then click two points on the map to measure any distance. Drag the handles to adjust. Useful for checking distances between trees or from a tree to the polygon edge.
💡 Click any numbered tree marker on the map for a popup showing its tree number, UID, coordinates, and inside/outside status.
5 — Editing Polygons
Starting an edit
Click ✏ Edit Polygon to enter edit mode. Yellow vertex markers appear at each corner of the polygon. Drag any marker to reshape the boundary. The polygon updates live as you drag.
Applying or discarding
When satisfied, click ✓ Apply to commit the new shape — the analysis re-runs automatically with the corrected boundary. Click ✗ Cancel to discard all changes and restore the original polygon.
⚠ Applying an edit only updates the polygon in your browser session. To write it back to Fieldforce permanently, you must click 💾 Save to Fieldforce after applying.
6 — Saving to Fieldforce
Write-back
Click 💾 Save to Fieldforce to write the current polygon boundary back to the ccs_polygongeojson_plot field on the plot record in Dataverse. A confirmation dialog shows the plot name and GUID before saving.
The button turns green and shows ✓ Saved to Fieldforce! on success, then resets after 4 seconds.
⚠ This action permanently overwrites the polygon in Fieldforce. It cannot be undone from this tool. Make sure the polygon is correct before saving.
Permissions required
You must have write access to the ccs_plot table in Dataverse. If save fails with a 403 error, contact your Fieldforce administrator to request write permissions.
7 — Plot Catalog
Opening the catalog
Click ⊞ Plot Catalog in the header (visible after connecting to Fieldforce). The catalog shows all plots as a visual grid of satellite maps with their polygon boundaries and trees plotted.

Filters & search
Use the dropdown to show All plots, With polygon ✓, or No polygon ✗. Use the search box to filter by plot name. Pagination controls (← Prev / Next →) navigate between pages.
Expanding a card (Lightbox)
Click any card to open a large lightbox view of that plot with full zoom and pan, both RefCenter (blue) and MeasuredCenter (purple) markers, tree dots, and compliance percentage. Click Open Plot → to load it in the main reviewer, or ✕ to close back to the catalog.
💡 The card footer shows trees inside vs total (e.g. 14/15 inside — 93%) colour-coded green/amber/red. Trees data loads once when the catalog opens and is shared across all cards.
8 — PDF Export
Single plot PDF
After running analysis on a plot, click ⬇ Download PDF to export a single-page report for that plot. The PDF includes the satellite map with polygon and tree markers, all measurements, and tree list.
Batch PDF
Add multiple plots to the batch queue using the + Add to Batch button. The batch counter in the header shows how many plots are queued. Click ⬇ Download All PDF to generate a multi-page PDF with one page per plot.
Plots can be removed from the batch queue individually using the ✕ button next to each entry in the batch list.
💡 Batch PDF is useful for generating a review pack for a full set of plots before a field visit — load each plot, verify it looks correct, add it to batch, then download all at once.

10 — How It Works
What is a plot polygon?
A plot polygon is a GPS boundary drawn around a registered planting plot — a digital fence tracing the outline of the plot area on a map. The system checks whether each tree's GPS pin falls inside or outside that fence.
Inside vs Outside
The system uses a point-in-polygon algorithm: for each tree coordinate, it checks whether the point lies inside the polygon boundary. Trees inside are green; trees outside are red.
What causes trees to appear outside?
Most common: the polygon is in the wrong place. If a plot polygon was drawn incorrectly or is offset from its true location, trees with accurate GPS coordinates will fall outside. The trees are correct — the polygon needs correcting.
Second cause: GPS error at planting. Occasionally a tree's GPS coordinate was recorded with significant error. If only one or two trees are outside while the rest are inside, this is more likely the cause than a boundary issue.
RefCenter vs MeasuredCenter offset
A large distance between the RefCenter (survey GPS) and MeasuredCenter (field officer GPS) often indicates the polygon was drawn at the survey point but trees were planted at a different physical location — or vice versa. Use this as a diagnostic signal before deciding which boundary to move.
💡 When every tree in a plot is outside the polygon, it almost always means the polygon needs to be redrawn — not that all the trees are in the wrong place.
